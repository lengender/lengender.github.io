实际用户ID、有效用户ID和设置用户ID

发表于 2017-06-19 | 分类于 Linux

实际用户ID、有效用户ID和设置用户ID是linux经常遇到的概念，现作如下整理：

RUID：用于在系统中标识一个用户是谁，当用户使用用户名和密码登录后，unix系统就确定了它的RUID

EUID：用于系统决定用户对系统资源的访问权限，通常情况下等于RUID

SUID：用于对外开放权限，RUID和EUID是跟用户绑定，SUID跟文件绑定

文件的访问权限包括读写和执行，判断某个进程对文件有何权限时，内核会将非超级用户进程的EUID和所有者ID进行比较，当然，也可能需要比较有效组ID，这关系到具体的权限测试方法，此处不再赘述。

文件的所有者ID，即为创建文件的进程的有效ID作为文件的所有者ID

一般情况下，进程的有效用户ID被设置成执行该进程的实际用户ID，但有的时候，一个进程可能要去执行其它用户创建的文件，这时，该进程的EUID和该文件的所有者ID是不同的，但是如果该文件设置了”设置用户ID”或者”设置组ID”位，那么该进程在执行该文件时，就会将进程的有效用户ID临时更改为文件的所有者ID

SUID是由exec函数复制有效用户ID得来的，所以说SUID是进程EUID的副本，为什么要保留进程有效ID的副本呢？ 刚才讲到文件设置设置用户ID位时，内核会将执行进程的有效用户ID临时更换为文件所有者ID，执行完该文件后需要恢复原来的有效用户ID，所以需要事先保留一个副本。
